Exercício de Operadores - Relacionais, Lógicos e Ternário.

Tipos de operadores

  • Aritméticos
  • Atribuição
  • Relacionais
  • Lógicos
  • Ternário

Relacionais

  • 5 > 2 True
  • 7 < 4 False
  • 8 >= 8 True
  • 9 <= 7 False
  • 5 == 5 True
  • 4 != 4 False

Relacionais de identidade

  • 5 == 5 True
  • 5 == '5' True
  • 5 === '5' False
  • O igual == olha somente o valor e não o tipo.
  • Os 3 iguais === perguntam se é identico, valor e tipo, então neste caso em específico o número 5 é diferente da string '5' FALSE.

Operadores Lógicos

  • ! Negação
  • && Conjunção
  • || Disjunção

Rodando exemplos no nodejs

  • > var a = 5 (undefined)
  • > var b = 8 (undefine)
  • > true && false (false)
  • > true && true (true)
  • > false || false (false)
  • > true || false (true)
  • > true || true (true)
  • > a > b && b % 2 == 0 (false)

Exemplos

  • idade >= 15 && idade <= 17
  • Pergunta: A idade está entre 15 e 17?
  • estado ++ 'rj' || estado == 'sp'
  • Pergunta: O estado é RJ ou SP?
  • salario > 1500 && sexo !+ 'M'
  • Pergunta: O salário é maior que 1.500,00 e não é do sexo masculino?

Precedência de operadores lógicos

A sequência de solução para operadores lógicos será sempre primeiro o "negação / não", depois o "&& / E e por último o "|| / ou".

  • Negação ! (Não)
  • Conjunção && (E)
  • Disjunção || (Ou)

Operador Ternário

  • ?
  • :
  • teste ? true : false

Obs: É considerado ternário quando temos os dois dentro de uma expressão (se ou sen não).

Exemplo node:

  • > media >= 7 ? ' Aprovado ' : ' Reprovado '
    ' Reprovado '

  • > var x = 8
    undefined
    > var resultado = x % 2 == 0 ? 5 : 9
    undefined
    > resultado
    5

Lembrando a ordem de precedência para operadores aritméticos e de atribuição.

  • Parênteses ()
  • Potência **
  • Matemáticos * / % (dependendo de quem vier primeiro à esquerda)
